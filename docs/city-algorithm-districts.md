# 城市算法街区 — 完整需求文档

## 一、整体架构

| 项目 | 要求 |
|------|------|
| 城市风格 | 半写实夜景，emissive 材质为主，不用 pointLight |
| 算法设施 | 6 个真实城市场景，分散在城市各角落 |
| 相机 | 切换算法时跳到对应设施，不做 useFrame 回弹 |
| 可视化 | 算法演示是每个设施的主角 |
| 性能 | 0 NaN，不用 pointLight，不用 GLB 行人 |

## 二、6 个算法设施

| # | 算法 | 设施 | 位置 | 主色调 |
|---|------|------|------|--------|
| 1 | 排序×5 | 邮件分拣中心 | x=-32, z=-18 | 橙/棕 |
| 2 | 搜索×4 | 图书馆 | x=-32, z=20 | 暖黄/木色 |
| 3 | 树×5 | 苗圃/植物园 | x=-10, z=34 | 绿色 |
| 4 | DP×6 | 物流规划中心 | x=0, z=-10 | 蓝色 |
| 5 | 字符串×4 | 广告招牌店 | x=32, z=-18 | 紫色 |
| 6 | BFS/DFS | 消防站 | x=32, z=4 | 红色 |

## 三、每个设施的结构

```
XxxFacility（group position=[x, 0, z]）
├── 底板（该设施特有的地面材质/颜色）
├── 主体建筑/设备（体现算法原理的真实设施）
├── 算法可视化元素（节点/方块/指针等）
├── 状态标签/看板（显示算法执行步骤）
├── 装饰物（灯、围栏、植物等，不穿模）
└── 说明面板 + 🔍全屏按钮
```

### 设施 1：邮件分拣中心（排序×5）

```
┌─────────────────────────┐
│  ┌─传送带─────────────┐  │
│  │ 📦 → 📦 → 📦 → 📦 │  │
│  └────────────────────┘  │
│  [东区] [西区] [南区] [北区]  ← 分拣箱
│  工人把包裹从传送带放入对应箱子  │
│  朴素=一个一个试，优化=用优先级 │
└─────────────────────────┘
```

- 底板：灰色工厂地面
- 传送带：长条平台 + 移动的包裹方块
- 分拣箱：4个不同颜色的箱子
- 看板：显示当前比较/交换步骤

### 设施 2：图书馆（搜索×4）

```
┌─────────────────────────┐
│  📚📚📚📚📚📚📚📚📚📚  │  ← 书架（已排序）
│        🔍               │  ← 搜索指针
│  [目录台] [借书处]        │
│  线性=从头翻，二分=从中间翻  │
└─────────────────────────┘
```

- 底板：暖色木地板
- 书架：排列的书籍方块（已排序状态）
- 指针：left/right/mid 三个发光标记
- 目录台：显示搜索过程

### 设施 3：苗圃/植物园（树×5）

```
┌─────────────────────────┐
│     🌳 (根)              │
│    / \                  │
│  🌿  🌿 (子树)           │
│  /|  |\                 │
│ 🌱🌱 🌱🌱 (叶)          │
│  [品种标签] [高度标记]     │
└─────────────────────────┘
```

- 底板：绿色草坪
- 树木：不同大小的树模型，连线表示父子关系
- 标签：节点名称 + 高度/平衡因子
- 围栏：花园围栏

### 设施 4：物流规划中心（DP×6）

```
┌─────────────────────────┐
│  ┌─DP表格─────────────┐  │
│  │ 0  1  2  3  4  5   │  │
│  │ 1  ■  ■  ■  ■  ■   │  │ ← 当前计算的格子
│  │ 2  ■  ■  ★  ■  ■   │  │ ← 最优解
│  └────────────────────┘  │
│  路线图 + 依赖箭头        │
└─────────────────────────┘
```

- 底板：蓝色网格地板
- DP表格：3D方块矩阵，颜色深浅表示值
- 箭头：依赖关系连线
- 看板：当前状态 + 公式

### 设施 5：广告招牌店（字符串×4）

```
┌─────────────────────────┐
│  文本: ABCABDABCABD      │
│  模式:   ABCABD          │
│       ↑↑↑ 匹配中         │
│  [匹配成功] [回退]        │
│  KMP=跳过已知不匹配位置    │
└─────────────────────────┘
```

- 底板：紫色商业区地面
- 字符方块：排列的文字方块
- 滑动窗口：模式串在主串上移动
- 匹配高亮：匹配的字符发光

### 设施 6：消防站（BFS/DFS）

```
┌─────────────────────────┐
│  🚒 消防车               │
│  💧 BFS: 水逐层扩散       │
│  🔥 火源                 │
│  🚶 DFS: 消防员深入搜索    │
│  [楼层平面图] [搜索路径]    │
└─────────────────────────┘
```

- 底板：深色应急中心地面
- 水流扩散：同心圆/波纹表示BFS
- 搜索路径：深色线条表示DFS
- 消防员/消防车模型

## 四、空间规则（不能违反）

| 规则 | 具体范围 |
|------|---------|
| 主路 | z ∈ [-4, +4]，x ∈ [-45, +45] |
| 主路人行道 | z ∈ [-7, -4] 和 [+4, +7] |
| 左侧路 | x ∈ [-22, -18]，z ∈ [-38, +38] |
| 右侧路 | x ∈ [+18, +22]，z ∈ [-38, +38] |
| 路人人行道 | x ∈ [-24, -22]、[-18, -16]、[+16, +18]、[+22, +24] |
| Dijkstra 公园 | x ∈ [20, 40]，z ∈ [10, 26] |
| 配送站 | 中心 [3, 0, -3]，旋转 π/2 |
| 算法状态牌 | [-31, 5.4, -21] |
| 社区节点行 | z = -30，x ∈ [-34, +36] |

**设施必须避开以上所有区域，每个设施加 exclusion zone。**

## 五、视觉规则

| 规则 | 说明 |
|------|------|
| 不穿模 | 建筑/设施不能与道路、其他建筑重叠 |
| 正对主场景 | 相机正对设施主体，不被其他建筑遮挡 |
| 主场景不被遮挡 | exclusion zone 要足够大，确保相机视角内无遮挡 |
| 说明文字 | 精简、字号 ≥0.42、有描边 |
| 🔍全屏 | 2D 固定在屏幕正中，不跟视角动 |
| 每个设施不同色调 | 一眼能区分是哪个算法区域 |

## 六、交互规则

| 规则 | 说明 |
|------|------|
| 相机跳转 | 切换算法时瞬间跳到对应设施位置 |
| 无回弹 | 去掉 useFrame lerp |
| 看板实时更新 | 显示当前算法步骤、高亮节点 |
| 动态元素 | 路径/指针/包裹等有动画 |

## 七、代码改动清单

| 文件 | 改动内容 |
|------|---------|
| `CityScene.tsx` | 6 个新 district 组件 + exclusion zone + DISTRICTS 条件 + 分发 |
| `generateBlocks()` | 加 6 个 exclusion zone |
| `CityAlgorithmProjection` | 按 algorithmId 分发到对应设施 |
| `SceneFrame.tsx` | 不改（overlay 已通用） |
| `playback-store.ts` | 不改（showHelp 已有） |

## 八、完成后必须

1. **截图检查** — 每个设施切换后截图，确认：
   - 主场景完整可见，不被遮挡
   - 相机正对设施主体
   - 算法可视化清晰
   - 说明文字可读
2. **TypeScript 编译通过**
3. **0 NaN 错误**
4. **不穿模、不上路**

## 九、不能做

| 不能做 | 原因 |
|--------|------|
| 行人用 GLB | 白色 T-pose |
| 用 pointLight | 性能 |
| 建筑放路上 | 穿模 |
| 设施之间无间距 | 视觉混乱 |
| 相机被其他建筑挡 | 看不见主场景 |
| 全屏面板跟视角动 | 体验差 |

## 十、实现顺序

1. 排序分拣中心（最简单，柱状条+传送带，视觉冲击强）
2. 图书馆搜索区（数组+指针，和排序类似）
3. 苗圃区（节点+连线，和 Dijkstra 类似）
4. 物流规划中心（表格方块，需要依赖箭头）
5. 广告招牌店（字符方块+滑动窗口）
6. 消防站（图+队列/栈看板）
